# 体验重构计划

## 背景

当前 ShowMe 已经能把输入内容用大字号展示出来，但交互模型仍然偏向“一个大号输入框”。在家庭作业场景里，这会带来几个明显问题：

- 孩子观看时，展示区仍然是可编辑输入框，光标、键盘和输入法状态会干扰抄写。
- 页面加载后自动聚焦输入区，移动端容易直接弹出键盘。
- iOS 下输入框可能触发自动填充、密码键盘或输入法栏等系统行为。
- 展示范围固定，桌面上不够满屏，手机横竖屏下也不稳定。
- 设置面板浮在右上角，容易遮挡文字。
- 字号适配只跟窗口变化有关，没有真正跟输入内容变化同步。

这次重构的目标是把 ShowMe 从“可输入的大字框”调整为“孩子观看优先、家长编辑辅助”的抄写屏。

## 重构目标

### 核心目标

- 默认状态不弹键盘。
- 默认状态不出现可编辑光标。
- 展示文字尽量占满可用屏幕。
- 设置和编辑操作不遮挡孩子正在抄写的内容。
- 家长可以快速进入编辑、完成编辑、清空内容、调整字号和主题。

### 非目标

- 不做账号系统。
- 不做云端保存。
- 不做复杂学习管理。
- 不引入重型 UI 框架。
- 不在本轮做远程多设备同步。

## 新交互模型

### 展示态

展示态是默认状态，面向孩子。

- 页面主体是不可编辑的文字展示区。
- 没有输入框边框、光标、拼写检查标记或输入法干扰。
- 文字居中或按短文模式排版，尽量保持清楚、稳定。
- 屏幕边缘保留极少量控制按钮，例如“编辑”和“设置”。

### 编辑态

编辑态面向家长。

- 点击“编辑”后打开底部编辑面板或全宽抽屉。
- 编辑面板内使用 `textarea`。
- 只有编辑态才主动聚焦输入框。
- 输入框明确配置移动端输入属性，降低 iOS 自动填充和错误键盘类型的概率。
- 点击“完成”后收起编辑面板，回到展示态。

### 设置态

设置不应压住主要文字。

- 设置建议放入底部工具栏、底部抽屉或与编辑面板合并。
- 常用设置放第一层：字号、主题、清空。
- 低频设置放第二层：字体、颜色。
- 设置完成后能快速回到展示态。

## 组件结构建议

建议把当前单文件大组件拆成几个轻量组件：

- `App`：保存全局状态，协调展示态、编辑态、设置态。
- `DisplayStage`：只负责渲染孩子看的文字。
- `EditorPanel`：只负责家长输入。
- `Toolbar`：负责编辑、完成、清空、设置入口。
- `SettingsSheet`：负责字号、主题、字体、颜色。

第一轮可以先不拆文件，只在 `App.tsx` 里按这些边界组织函数和 JSX。等交互稳定后再拆文件，避免同时做太多结构变化。

## 技术方案

### 展示区

用普通元素替代展示态的 `textarea`：

```tsx
<main className="display-stage">
  <div className="display-text">{text || placeholder}</div>
</main>
```

这样展示态不会触发键盘，也不会被系统当作输入控件。

### 编辑区

编辑态继续用 `textarea`，但只存在于编辑面板中：

```tsx
<textarea
  value={text}
  onChange={(event) => setText(event.target.value)}
  autoComplete="off"
  autoCorrect="off"
  autoCapitalize="none"
  spellCheck={false}
  inputMode="text"
  enterKeyHint="done"
  name="showme-copy-text"
/>
```

这些属性不能百分百控制 iOS 行为，但可以减少 Safari、输入法、密码管理器把它误判为特殊输入框的概率。

### 视口和布局

- 使用 `min-height: 100svh` 或 `100dvh`，减少移动端地址栏导致的高度跳动。
- 展示区使用全屏弹性布局，而不是固定 `800x600`。
- 工具栏使用固定高度，并让展示区为它预留空间。
- 避免设置面板覆盖文字主体。

### 字号策略

第一轮不做复杂自动排版，先做稳定策略：

- 保留手动字号档位。
- 输入内容变化时同步应用字号。
- 单字和短词允许非常大。
- 短文模式使用更小字号和更舒适行高。

第二轮再考虑自动适配：

- 根据字符数选择默认字号。
- 根据展示区尺寸动态计算最大可读字号。
- 区分“单字模式”“词语模式”“短文模式”。

## 分阶段实施

### 阶段 1：稳定交互骨架

目标：解决聚焦、键盘、展示区和设置遮挡的主要问题。

- 新增 `text` 状态，把输入内容从非受控 `textarea` 改成受控状态。
- 移除页面加载后的自动 `focus()`。
- 展示态改成不可编辑 `div`。
- 新增“编辑/完成”状态切换。
- 编辑态才渲染或聚焦 `textarea`。
- 给 `textarea` 补齐移动端输入属性。
- 展示区改为全屏优先布局。
- 设置从右上角浮层改到底部区域或与编辑面板合并。

验收：

- 打开页面不会自动弹键盘。
- 点击“编辑”才弹键盘。
- 点击“完成”后文字仍显示，但键盘收起，光标消失。
- 孩子抄写时设置面板不遮挡主体文字。

### 阶段 2：面向抄写优化

目标：让不同长度内容更适合孩子照抄。

- 增加“单字”“词句”“短文”展示模式。
- 单字模式：超大字号、居中展示。
- 词句模式：大字号、适度换行。
- 短文模式：较小字号、舒适行高、段落清楚。
- 增加一键清空。
- 增加占位提示，说明可输入字、词或短文。

验收：

- 单字足够大。
- 一句话不会挤到边缘。
- 短文可以稳定阅读和抄写。
- 家长能快速清空并输入下一段。

### 阶段 3：高级但仍轻量的辅助

目标：在不破坏核心路径的前提下提升长期使用体验。

- 本地历史记录，默认只保存在当前设备。
- 常用内容模板，例如错字订正、今日抄写。
- 可选田字格、米字格或下划线背景。
- 全屏模式。
- 远期再考虑多设备同步。

验收：

- 新功能不会让默认界面复杂化。
- 家长仍然能在几秒内完成输入和展示。
- 孩子观看区仍然安静。

## 风险和注意点

- iOS 键盘行为不能完全由网页控制，只能通过减少输入控件暴露、正确设置属性和避免自动聚焦来降低问题概率。
- 如果展示态和编辑态切换太频繁，家长会觉得麻烦；因此“编辑/完成”按钮必须明显、顺手。
- 自动字号适配容易变得不可预测，第一轮应优先稳定和手动可控。
- 设置项不能一次性堆太多，否则会偏离“临时给孩子看清楚”的产品目标。

## 第一轮推荐改动清单

1. 改 `index.html` 标题为 `ShowMe`，补中文语言属性。
2. 移除 `textareaRef.current.focus()` 的启动自动聚焦。
3. 增加 `text`、`isEditing` 两个核心状态。
4. 展示态用 `div` 渲染文本。
5. 编辑态用底部面板承载 `textarea`。
6. 给编辑 `textarea` 增加移动端输入属性。
7. 把设置入口改成底部工具栏或底部设置面板。
8. 展示区使用 `100svh`/`100dvh` 和响应式安全边距。
9. 加占位提示和一键清空。
10. 跑桌面、窄屏和移动模拟视口检查。
