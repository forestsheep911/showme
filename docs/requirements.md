# 需求说明

## 核心场景

家长在孩子写作业时，打开 ShowMe，输入孩子需要看的字、词、句子或短文。ShowMe 用大字号、清楚字体展示内容，孩子照着屏幕完成抄写、订正或练习。

## 用户故事

### 生字展示

作为家长，我希望输入一个孩子不会写的汉字，让它以很大的字号显示出来，这样孩子可以照着写。

验收标准：

- 打开页面后输入框自动可用。
- 单个汉字显示足够大。
- 字体清晰，不被按钮或面板遮挡。

### 词语抄写

作为家长，我希望输入一个词语或几个词，让孩子看着抄写。

验收标准：

- 多个字可以自然换行或保持可读。
- 字距、行距适合抄写。
- 屏幕上没有太多干扰元素。

### 短文抄写

作为家长，我希望输入一小段短文，让孩子看着屏幕抄写，而不是让我重新手写一遍。

验收标准：

- 多行文本可输入。
- 文本区域支持滚动或合理缩放。
- 字号可以按短文长度调整。

### 晚上或低光环境

作为家长，我希望切换深色模式，避免屏幕太刺眼。

验收标准：

- 可以在浅色和深色主题之间切换。
- 切换主题后文字颜色自动保持可读。

### 字体和颜色调整

作为家长，我希望选择更适合孩子阅读的字体和颜色。

验收标准：

- 可以选择常见 CJK 字体。
- 可以选择高对比度文字颜色。
- 设置操作完成后可以回到专注展示状态。

## 当前已实现能力

- 大字号文本输入和展示。
- 字号档位切换。
- 字体选择。
- 文字颜色选择。
- 浅色/深色主题。
- 点击输入区时关闭设置面板。
- 静态网站构建和 Azure Static Web Apps 部署配置。

## 体验问题和改进机会

- 页面标题仍是默认 Vite 标题，没有体现产品用途。
- 设置入口使用 emoji 齿轮，风格不够稳定。
- 默认没有占位提示，新用户第一次打开可能不知道直接输入即可。
- 字号缩放只在窗口变化时处理，没有随着输入内容实时调整。
- 设置面板会占据右上角空间，对小屏幕和孩子抄写可能有干扰。
- 没有全屏/专注模式。
- 没有清空按钮、复制粘贴辅助、历史记录等家长高频操作。
- 没有针对“短文抄写”的行宽、行高、分段展示优化。

## 设计约束

- 优先保证本地输入和展示，不应强依赖后端。
- 默认不需要账号。
- 默认不保存孩子作业内容，除非未来明确设计本地历史记录。
- 界面文案应面向家长，避免复杂学习产品术语。
- 儿童阅读区应保持安静、清楚、高对比度。
